# Migrate to pnpm Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use plan-execute to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the entire PaceForge project from npm to pnpm package manager, updating all configuration files, CI/CD workflows, and documentation.

**Architecture:** Replace npm with pnpm as the package manager by updating package.json engines, CI/CD workflows, documentation, and generating a new pnpm-lock.yaml while removing package-lock.json.

**Tech Stack:** pnpm (fast, disk space efficient package manager), existing PaceForge stack (React, Vite, TypeScript, etc.)

---

## File Structure

### Files to Modify:
- `package.json` - Update engines field from npm to pnpm
- `.github/workflows/ci.yml` - Update Node.js setup and cache to use pnpm
- `.husky/pre-commit` - Update npm command to pnpm
- `README.md` - Update prerequisites and all npm command references
- `docs/architecture.md` - Update all npm references to pnpm in documentation
- `.gitignore` - Ensure package-lock.json is explicitly ignored

### Files to Create:
- `pnpm-lock.yaml` - Auto-generated by pnpm install command

### Files to Delete:
- `package-lock.json` - Remove after successful pnpm migration

## Rollback Procedure (if needed)

If migration fails and rollback is required:

```bash
# Remove pnpm lock and node_modules
rm -rf node_modules pnpm-lock.yaml

# Restore package-lock.json from git
git checkout HEAD -- package-lock.json

# Reinstall with npm
npm install

# Revert package.json engines change
git checkout HEAD -- package.json

# Revert CI/CD changes
git checkout HEAD -- .github/workflows/ci.yml

# Revert documentation changes
git checkout HEAD -- docs/architecture.md

# Revert README changes
git checkout HEAD -- README.md

# Revert pre-commit hook changes
git checkout HEAD -- .husky/pre-commit

# Revert .gitignore changes
git checkout HEAD -- .gitignore
```

## Notes for Implementers

1. **Why pnpm?** pnpm is faster, more disk-efficient, and creates stricter dependency management than npm. It uses symlinks and content-addressable storage.

2. **Breaking Changes:** None expected for this project. All npm scripts are compatible with pnpm.

3. **Team Communication:** After migration, update team documentation and communicate the change to all developers.

4. **CI/CD:** The GitHub Actions workflow will automatically use pnpm after this change. No manual intervention needed.

5. **IDE Integration:** Most IDEs (VS Code, WebStorm) automatically detect pnpm when `pnpm-lock.yaml` is present.

6. **Husky Hooks:** The pre-commit hooks will continue to work as they use npm scripts, which pnpm can execute.

7. **npm-run-all:** This package remains compatible with pnpm and continues to work as expected.
